# Second Brain — Alpha Release Action Plan (1-2 weeks)

## Phase 1: Critical Fixes & Security (Days 1-2)

### 1.1 Remove hardcoded Gemini API key
- **File:** `lib/main.dart:16`
- **Action:** Remove the hardcoded `AIzaSy...` key. API key entry should only happen via Settings screen (already exists). If no key is set, disable cloud LLM with a clear prompt to configure it.

### 1.2 Replace placeholder encryption in backup service
- **Files:** `lib/features/backup/cloud_backup_service.dart`, `lib/features/settings/settings_screen.dart`
- **Action:** Replace XOR "encryption" with AES-256-GCM using the existing `SecureKeyService` via MethodChannel. This reuses the security infrastructure already in place for the database.

### 1.3 Add global error handling & logging
- **File:** `lib/main.dart`
- **Action:** Wrap the app in a `Zone` that catches unhandled errors and reports to Firebase Crashlytics (already configured). Add a simple `Logger` utility class for structured debug/info/error logging throughout the app.

## Phase 2: Core Flow Polish (Days 3-5)

### 2.1 Add empty states to all screens
- **Overlay tab:** When no analysis done, show a clear "Tap the floating brain to capture" CTA
- **Chat/Search tab:** When no memories exist, show "Your second brain is empty. Start by capturing content from any app."
- **Settings:** Already decent — minor polish only

### 2.2 Add memory management to Chat screen
- **File:** `lib/features/chat/chat_screen.dart`
- **Action:** Add swipe-to-delete and tap-to-view-detail for memory entries. Add a "Delete all" option in the app bar. This uses the existing `MemoryRepository.deleteMemoryEntry()` and `deleteAllEntries()`.

### 2.3 Improve overlay result display
- **File:** `lib/features/overlay/overlay_screen.dart`
- **Action:** Show loading indicators during capture/analysis phases. Show clear error states if capture fails or API key is missing. Make the keyword chips tappable to filter/search by keyword.

### 2.4 Sync settings between Dart and Kotlin
- **Issue:** SharedPreferences is accessed independently from both sides, leading to potential race conditions
- **Action:** Add a `SettingsChannel` (new MethodChannel or piggyback on existing one) so Kotlin-side reads from the Dart `SettingsService` as single source of truth. Key settings to sync: `scanOnLike`, `scanOnSave`, `enableBuffer`.

## Phase 3: User Onboarding (Days 5-6)

### 3.1 Build a simple onboarding flow
- **New files:** `lib/features/onboarding/onboarding_screen.dart`, `lib/features/onboarding/onboarding_page.dart`
- **Action:** 3-page walkthrough:

### 3.2 First-launch experience
- **File:** `lib/app.dart`
- **Action:** Check `onboardingComplete` flag on app start. If false, show `OnboardingScreen` instead of `MainShell`.

## Phase 4: AI Quality Improvements (Days 6-8)

### 4.1 Add rate limiting to cloud LLM calls
- **File:** `lib/features/analyze/cloud_analysis_service.dart`
- **Action:** Implement a simple token bucket rate limiter (max 20 requests/minute for Gemini, 10 for OpenAI, 10 for Anthropic). Show user-facing cooldown message when rate limit is hit.

### 4.2 Improve analysis prompts
- **File:** `lib/features/analyze/cloud_analysis_service.dart`
- **Action:** Better system prompts for each mode:
  - **Summarize:** "You are a concise summarizer. Provide a 2-3 sentence summary of the captured screen content."
  - **Explain:** "Explain the content on this screen in simple terms as if teaching a beginner."
  - **Translate:** Use existing targetLanguage parameter but improve prompt structure.
- Add a **"Key Takeaways"** mode that extracts actionable items.

### 4.3 Add context-aware analysis
- **File:** `lib/features/overlay/overlay_notifier.dart`
- **Action:** Pass the `currentApp` package name as context to the cloud LLM prompt so it can tailor analysis (e.g., "This is from WhatsApp" → focus on conversation extraction).

### 4.4 Add "Key Takeaways" as 4th analysis mode
- **New values in enum/selector:** Add to analysis mode picker in `overlay_screen.dart`
- **Action:** Extract bullet-point takeaways from captured content. Uses existing `CloudAnalysisService` with new prompt.
